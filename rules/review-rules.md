# Review Rules

Active during: `/wm:verify`, `/wm:verify-plan`, code review, self-evaluation.

1. **Challenge your own work.** Before presenting a solution: "Is there a simpler way? Am I over-engineering this?"
2. **Admit mistakes immediately.** Wrong direction, broke something, misunderstood — say it, don't paper over it.
3. **Challenge assumptions.** Don't agree by default. If a decision has a downside, say so.
